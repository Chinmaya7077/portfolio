PROJECT: EZWallet - Autonomous Solana Trading System

Tech Stack: Python, FastAPI, Solana, gRPC, Redis, ClickHouse, PostgreSQL

Role: Sole backend developer responsible for designing the trading engine, writing all FastAPI services, setting up gRPC integrations, and managing Redis/ClickHouse/PostgreSQL data layers.

Company: Built at HCL Technologies

What I built:
- An autonomous trading system that manages Solana token positions end to end on PumpFun and PumpSwap, executing buys and sells in the same on-chain slot as the trigger.
- Integrated direct gRPC validator streams to evaluate user-defined triggers on every on-chain tick, replacing fixed-interval API polling with real-time event evaluation.
- Implemented unlimited take-profit and stop-loss levels per order, trailing stop-loss tracking peak price, and a timeout watchdog that auto-closes stale positions without user action.
- Added real-time rug protection by monitoring pool liquidity continuously and auto-exiting positions the moment liquidity dropped below per-order thresholds.
- Engineered a parallel RPC submission layer with smart retry (3x buy, 10x sell, exponential backoff), achieving same-slot landing on critical sell transactions.
- Stored encrypted user wallets and order state in PostgreSQL, used Redis for position state and publish-subscribe, and ClickHouse for trade analytics and settlement price extraction.
- Exposed REST endpoints supporting price triggers (above, below, approximately equal), AND or ANY logic, and GTC, IOC, and GTT time-in-force options for flexible order placement.
